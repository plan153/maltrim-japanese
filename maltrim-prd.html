① 제품 개요 (Product Overview)

🎌 한 줄 정의 (Elevator Pitch)

말트임은 왕초보 한국인 학습자가 말부터 트이게 해주는 일본어 앱으로,
원어민 TTS + AI 발음 교정 + 에빙하우스 스케줄을 결합해 "30일 안에 여행 일본어 완주"를 보장합니다.

🏗 현재 구현 상태
  • 단일 HTML SPA — GitHub Pages 배포
  • Vercel Serverless (Node 20) 백엔드
  • Neon PostgreSQL (에빙하우스 진도 DB)
  • Upstash Redis (API 응답 캐싱)
  • Claude Sonnet/Haiku (예문 생성 + 발음 피드백)
  • Google Neural2 TTS 사전 생성 MP3
🎯 제품 포지셔닝
  • 경쟁사 대비: Duolingo(게임화 우선) vs. 말트임(발화 우선)
  • 차별화: STT 실시간 채점 + FFD 반응속도 계측
  • 타깃 포지션: "말하기 특화 일본어" 카테고리 1위
  • 수익 모델: 프리미엄 구독 (뉴럴 TTS + AI 코치)
📐 핵심 원칙
  • 말하기 우선 (쓰기/읽기는 부가 기능)
  • ます형 선행 학습 (문법 설명 최소화)
  • 스트릭 기반 습관 형성
  • 오프라인 우선 (핵심 기능 네트워크 불필요)
  • 단일 세션 ≤ 15분 (마이크로 러닝)

② 사용자 문제 (User Problems)

핵심 가설: 한국 성인 학습자의 60%+가 일본어를 "읽고 쓰기"는 학습했지만, 실제 대화에서 첫 마디를 꺼내지 못한다. 말하기 훈련 없이 문법·어휘만 공부하기 때문이다.
#문제현재 해결 방법한계
P1 말하기 공포
"일본어로 말하면 발음이 이상할까 봐 두렵다"		 유튜브 섀도잉, 학원 피드백 없음 / 비용 높음
P2 지속 불가 학습
"매일 하려고 했는데 3일이면 포기한다"		 앱(Duolingo 등) 게임화만 있고 실제 발화 훈련 없음
P3 불명확한 진도
"내가 어느 수준인지 모르겠다"		 교재 페이지 진도 실제 발화 실력과 무관
P4 망각
"배운 것을 금방 잊어버린다"		 반복 암기 에빙하우스 스케줄 미적용
P5 여행/실용 목적 괴리
"JLPT 문제만 풀다가 실제 여행에서 말 못 했다"		 JLPT 교재, 회화 책 실전 발화 연습 부재

③ 목표 / 비목표 (Goals & Non-Goals)

✅ 목표 (In Scope)
  • 30일 커리큘럼 완주율 50% 이상 달성
  • 사용자당 주간 학습 세션 ≥ 5회
  • 첫 세션 완료율 70% 이상
  • 말하기 정확도 평균 80점↑ 달성 (4주 후)
  • 앱스토어 평점 4.5+ 유지
  • 월간 유료 전환율 5% 이상
  • iOS / Android 네이티브 앱 출시 (v2.0)
🚫 비목표 (Out of Scope)
  • JLPT 시험 대비 전문 콘텐츠 (≥ N2 레벨)
  • 일본어 → 한국어 번역 기능
  • 사용자 간 소셜 기능 (커뮤니티, 채팅)
  • 중국어·영어 등 타 언어 지원
  • 강사 연결 / 1:1 튜터링
  • 웹 RTC 실시간 화상 회화
  • 사용자 직접 커리큘럼 편집 기능
⚠️ 범위 경계선: "쓰기 연습"과 "읽기 전용 학습"은 보조 기능으로만 제공. 핵심 엔진은 항상 발화(STT) 기반이어야 한다. 이를 희석하는 기능 요청은 로드맵 외로 분류한다.

④ 대상 고객 (Target Users)

🧳

Persona A — "여행 준비 직장인" (Primary)

25~40세, 일본 여행 계획 중. 앱/게임/유튜브는 익숙하나 일본어 학습 경험 없음. 출퇴근 시간(15~20분)에 스마트폰으로 학습하고 싶다. 목표: 식당 주문·길 묻기·쇼핑 표현.

📱 스마트폰 헤비유저 ⏱ 하루 10~15분 🎯 여행 전 3~4주
💼

Persona B — "출장 비즈니스맨" (Secondary)

30~50세, 일본 거래처 미팅 예정. 기초 인사·감사 표현 필요. 이메일·영어는 가능하나 대화를 "존댓말"로 시작하는 것에 부담감을 느낌.

💳 지불 의향 높음 ⏱ 하루 5~10분 🗣 비즈니스 문체
🎌

Persona C — "문화 덕후" (Secondary)

20~35세, 애니·드라마·음악 팬. 자막 없이 일본어를 이해하고 싶고, 좋아하는 콘텐츠 속 표현을 직접 말해보고 싶다. 학습 동기 강하지만 지속성 약함.

🎮 게이미피케이션 선호 📱 일 30분+ 가능 💬 구어체 선호

⑦ 경계 사례 (Edge Cases)

🎙 STT / 오디오
  • no-speech: 4초 침묵 → 토스트 "소음 없는 곳에서 말씀해 주세요" + 재시도
  • not-allowed: 마이크 거부 → 인앱 설정 안내 바텀시트
  • audio-capture: 하드웨어 오류 → 마이크 확인 안내
  • STT 한국어 인식: lang=ja-JP 강제 고정; 한국어 결과 수신 시 점수 0처리
  • 장문 STT: 문장 길이 ≥ 30자 → 부분 매칭 허용
🌐 네트워크
  • 오프라인 진입: 서비스 워커 캐시 → 저장된 문장 학습 유지
  • DB 저장 실패: 로컬 큐 저장 → 온라인 복귀 시 일괄 sync
  • Vercel Cold Start: 첫 요청 ≥ 3s → 클라이언트 재시도 1회 자동
  • Redis 다운: graceful fallback → 캐시 없이 Claude 직접 호출
👤 사용자 행동
  • 백그라운드 전환: 녹음 중단 → visibilitychange 감지 후 자동 중지
  • 빠른 탭 연타: TTS 중복 재생 방지 → 재생 중 버튼 비활성
  • 진도 초기화 오작동: 이중 확인 다이얼로그 + 입력 검증
  • 처음부터 90점 이상: stage 빠른 진급 → stage 6 도달 후 장기 복습 모드
🤖 AI / 데이터
  • Claude 429 / 500: 재시도 2회 후 기본 피드백 텍스트로 대체
  • JSON 파싱 실패: parseJsonSafe 예외 → 오류 로그 + 폴백 예문 반환
  • score_bucket 오남용: bucket=0(점수 0~9)과 bucket=100 경계 처리
  • 긴 reference_text: 캐시 키 길이 제한 → SHA-256 해시로 단축
